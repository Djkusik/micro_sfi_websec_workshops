Command injection


Attack in which unsafe user input is passed to a system shell.
The attacker-supplied system commands could be executed on the system with the privileges of the vulnerable application.

Why it happens?


Probably some old application or part of specific internal system, because nowadays application which uses user input as part of a shell command is rarely seen.

How to exploit it?


Use special shell characters to escape from executed command and use own system command.

How to secure it?


Do not secure it, but also do not use user input as part of shell command.
